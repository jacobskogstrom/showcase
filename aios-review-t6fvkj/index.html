The 11 feature clusters — verdict at a glance

1 · Mobile command center daily keep custom

Telegram bridge, /todo, /run, /handover, /panic, /mirror. 243 msgs/14d. Sits exactly on the biggest official hard gap: phone → full repo + scripts. Nothing official comes close.

2 · Job-hunt engine daily keep custom

scan-jobs, recruiter-researcher, morning brief, contact-hub, Jobs CRM, email send. Needs Gmail send + Sheets write — both officially impossible. LinkedIn legs → Claude in Chrome.

3 · Personal KB in git daily already official

contacts.md, decisions log, todos, data-map + Claude Code. This IS the official pattern (CLAUDE.md + repo memory). Nothing to migrate — you're on it.

4 · Scheduled intelligence fires daily keep local

Architect nightly, mail digests, tokenburner, scan-jobs. Local Task Scheduler beats cloud Routines: subscription-covered, sub-hourly, has gws auth. Gap: zero read-telemetry.

5 · Bitesize content factory weekly human keep + adopt Design

Backlog → 20:30 brief → answers → 01:00 build → push-live → Cloudflare. Answer-loop has no official twin. Claude Design can raise episode-page quality inside the pipeline.

6 · AI Dashboard unmeasured keep, prune + fix

12 tabs + 2 static pages, no telemetry, dead /settings link, risky promote flow. Officials can't serve live private data — but half the tabs may have zero viewers. Instrument, then prune.

7 · Voice testing only migrate to app

Custom :8080 + tunnel + nightly context rebuild + ~25 commits/60d — but logs show only test chats. Claude app voice mode does conversational voice for free, zero maintenance.

8 · Media generation occasional keep, freeze growth

kie.ai images/video via /media, /visual2. Officials: no video path, no pipeline API. Keep Phase 1; Phases 2-3 stay frozen (zero use). App image gen for throwaway images.

9 · Silo product frozen stay frozen

23 use cases, ~9 lifetime client messages, zero willingness-to-pay. Frozen 6mo by your own 2026-07-04 decision. Officials have no multi-tenant path either — but the market said no.

10 · Dev & meta tooling weekly already official

Skills, git sync, /new-project, /audit, /level-up, dev modes. Skills system IS Claude Code native. Delete the 6 orphan scripts; install the missing /todo front doors.

11 · Growth experiments dormant kill / park

Mini App (unused), PromptLens (reviewed once), trend-to-money (never deployed), LinkedIn poster (OAuth-blocked). One rescue: LinkedIn posting via Claude in Chrome scheduled browser tasks.

Deep dives — how to do it my way vs the official way

1 · Mobile command center keep custom

As Jacob, away from the desk, I want to command my entire system from my phone so nothing waits for the desktop.

Your solution

  1. Message @Tojacobbot — listener pipes it into claude -p in the AIOS repo with full file, gws and script access
  2. /new fresh thread · /bg background task · /run whitelisted commands · /panic kill-switch
  3. Photos in, files out via SEND_FILE; handover to desktop with /handover

Official tools

  1. Claude app (iOS): chat + Memory + Projects; Gmail/GCal connectors read-only-ish; no repo, no scripts, no gws
  2. claude.ai/code from phone browser: cloud session on the GitHub repo — can edit files, but can't touch Hbot scripts, local auth, or Telegram
KEEP CUSTOM. This is the killer app and it sits on an official hard gap ("mobile capture into full repo context"). Honest note: install the Claude app anyway as the free disaster-recovery channel for when Hbot is down — its Memory makes it a decent degraded mode.

2 · Job-hunt engine keep custom

As Jacob, I want every job lead scanned, scored, tracked and followed up so the July 31 deadline is hit with volume and quality.

Your solution

  1. /scan-jobs — Platsbanken API, 14d window, dedupe vs applied-roles, cross-ref contacts → Telegram
  2. Paste a job ad → recruiter-researcher agent returns Go/No-Go vs about-me.md
  3. /contact-hub "lunch with X" → fans out to contacts.md + Google Contacts + Networking calendar + Jobs sheet in one confirm
  4. send_email.py sends as jacob@skogstrom.se; /morning-brief at 08:00; Jobs kanban on the dashboard

Official tools

  1. Claude app + GCal connector: ask "what's today" — decent brief, calendar R/W works
  2. Gmail connector: drafts only — you press send yourself
  3. CRM: no Sheets write — falls back to Project notes (unstructured) or manual sheet edits
  4. Claude in Chrome: LinkedIn triage/outreach in-browser on a schedule — this one is a real official win
KEEP CUSTOM — the engine needs email send + Sheets write, both official hard gaps. But adopt Claude in Chrome for the three LinkedIn pieces (triage, contact enrich, posting): it replaces two orphan scripts and the OAuth-blocked poster with an official, scheduled browser agent.

3 · Personal KB in git already official

As Jacob, I want my contacts, decisions and todos versioned, greppable and editable by my agent so context never evaporates.

Your solution

  1. Markdown files in the private AIOS repo: contacts.md, decisions/log.md, inbox/todos.md, data-map
  2. Claude Code reads/writes them in every session; git = history + rollback

Official tools

  1. Identical — this is the official Claude Code pattern (CLAUDE.md + repo files + auto-memory)
  2. claude.ai Memory adds an auto-synthesized slice for the app surface, but it's opaque and not diffable
ALREADY ALIGNED. No migration exists because you're standing on the official architecture. The app's cloud Memory is a complement (mobile degraded mode), not a replacement — your git KB stays the source of truth.

4 · Scheduled intelligence keep local

As Jacob, I want the system to brief me, digest mail and audit itself on schedule so I start each day pre-informed.

Your solution

  1. Windows Task Scheduler: architect 02:00, tokenburner 01:00, mail digest 12:00 + 19:00, scan-jobs 08:17, brief 08:00, Bitesize 20:30 + 01:00
  2. Each runs headless claude -p (subscription-covered) with local gws auth, pushes to Telegram

Official tools

  1. Claude Code Routines: cloud cron ≥1h, GitHub-repo access, fires with the PC off
  2. Can't reach gws auth or Telegram delivery without shipping secrets to the cloud; metered API on top of the subscription
KEEP LOCAL. Cheaper (subscription vs metered), finer-grained, and holds the local auth. Routines' one real edge is surviving Hbot downtime — worth a pilot for the repo-only architect nightly as disaster recovery, nothing else. The actual gap here isn't infra, it's measurement: instrument whether digests get read, then kill the unread ones.

5 · Bitesize content factory keep + adopt Design

As Jacob, I want ideas to become reviewed, publishable episodes overnight so the brand grows while I sleep.

Your solution

  1. /bitesize idea → backlog; 20:30 brief pings Telegram; you answer in numbered replies
  2. 01:00 build (opus) makes the episode page + shoot script + renders; morning handoff
  3. Vet in staging gallery → push-live.ps1 → Cloudflare Pages → n8n cross-posts video

Official tools

  1. Idea capture: Claude app Project — works but lands outside the pipeline
  2. Nightly build: a Routine on the GitHub repo could build pages in the cloud
  3. Episode pages: Claude Design with the design system read from your codebase — likely better pages than the template; export HTML back into showcase/
  4. The evening ask → verbatim answer loop: no official equivalent (needs Telegram)
KEEP THE LOOP — it's used (59% answer rate) and the answer-capture leg is unreplicable officially. One upgrade: pipe episode-page generation through Claude Design and keep publishing via Cloudflare.

6 · AI Dashboard keep, prune + fix

As Jacob, I want one screen that shows system health, work state and the job pipeline so I don't grep for status.

Your solution

  1. Next.js dashboard on the private network, served to the phone over Tailscale
  2. Live tabs over private data: Jobs (Sheet), Development (SQLite + claude CLI), Activity (session hooks), Todo, Map (iCloud), services health

Official tools

  1. Claude Design / Artifacts: generate + share static pages — no live server, no private-network data, and pushing PII to claude.ai contradicts why the repo is private
  2. No official product serves a live dashboard over local data. Period.
KEEP — officials genuinely can't do this. But the critic is right that it's an unmeasured 14-surface build: add a 1-line page-view log, wait 2 weeks, prune the tabs with zero views. Fix now: dead /settings link, PromptLens tab (feature is being retired), document the promote flow's all-diffs gotcha.

7 · Voice migrate to app

As Jacob, I want to talk to my assistant hands-free from anywhere.

Your solution

  1. /voicelink → custom voice server through a Cloudflare tunnel; Claude deep tier + Gemini Flash fast tier
  2. 06:00 context rebuild, calendar refresh, screen-router — ~25 commits of upkeep in 60 days

Official tools

  1. Claude app voice mode: push-to-talk conversational voice, reaches GCal/Gmail/Docs connectors, zero maintenance, no tunnel
  2. Doesn't drive the repo or scripts — but per your own logs, neither does the custom one (test chats only)
MIGRATE. The evidence is blunt: heavily built, not actually used for work. Claude app voice mode covers the real (conversational) use for free. Park the server, tunnel and 06:00 rebuild; revive only if "drive the PC by voice" becomes a demonstrated need, not a demo. This single move deletes a tunnel, a port, a nightly task and your biggest build sink.

8 · Media generation keep, freeze growth

As Jacob, I want on-brand images and video clips generated from a one-line ask.

Your solution

  1. /media routes NL to kie.ai (video + nano-banana images), logs to runs.md; /visual2 house-style PNGs + editable .excalidraw

Official tools

  1. Claude app image gen: fine for throwaway images, no pipeline API, no house-style consistency
  2. Video: no official path at all; Claude Design covers page-level visuals, not media assets
KEEP Phase 1 as-is (video has no official substitute; house-style needs the pipeline). Use app image gen for one-offs to save kie.ai spend. Phases 2 (editing) and 3 (posting) stay frozen — zero usage since built.

9 + 11 · Silos & growth experiments kill / park frozen

One-line treatment — these are dormant; full pipelines would be theater.

What exists

  1. Silo product: 23 UCs, ~9 lifetime messages, frozen 6mo by decision 2026-07-04 — stay frozen, revisit after a job is signed
  2. Mini App: built, never configured — kill
  3. PromptLens: silent rewrites, reviewed once — kill (keep on-demand /improveprompt)
  4. trend-to-money / whats-trending: 2 local demos — park; if revived, generate pages with Claude Design instead
  5. LinkedIn auto-poster: OAuth-blocked — replace with Claude in Chrome scheduled posting
  6. 9 orphan scripts — delete (git keeps them forever anyway)

Official angle

  1. Multi-tenant assistants: no official consumer path exists; Agent SDK = the same DIY you already did. The freeze was a market decision, not a tooling one
  2. Claude in Chrome is the one official product that rescues a dead feature here (LinkedIn)
PRUNE. This tail is ~40% of everything built. Deleting it costs nothing (git history keeps it), removes two ports and a mental tax, and makes the weekly /audit stop reporting the same orphans forever.

The panel — critic vs advocate, and where they agreed

The critic's core charge

"Build-to-use ratio is upside down, 26 days from the deadline. The dormant column holds Media Phases 2-3, the silos, the LinkedIn poster, the Mini App, PromptLens — none of it ladders to 'PO/AI role by July 31'."

"The silo product is a sunk-cost monument: ~9 lifetime messages against a 12-slot pool, nonce-binding and SOPS encryption."

"Bus factor 1: everything dies with one PC. The dashboard is a 14-surface building nobody's measured."

The advocate's core defense

"The Telegram bridge is the whole product — 243 messages in 14 days on exactly the hard gap officials can't fill. Migrate to the Claude app and you keep a chatbot, lose your operator."

"Email send, Sheets write, the 4-destination contact-hub fan-out, the live dashboard: every daily-use feature sits squarely on an official hard gap. That's a keep on merit, not sentiment."

"The honest tax is the dormant tail — prune it so 'keep' doesn't mean 'hoard'."

Where both landed (unprompted): keep the Telegram bridge, gws writes, git KB, job-hunt loop, Bitesize answer-loop · kill/park silo expansion, Media 2-3, Mini App, PromptLens, trend-to-money, the orphans · the only real fights were voice (critic won on evidence — test chats only) and how hard to prune the dashboard (split the difference: instrument first, then cut).
